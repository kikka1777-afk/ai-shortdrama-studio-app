# 给 Codex 的部署指令（复制粘贴即可）

项目位置：`D:\Drama Agent`

目标：推到 GitHub 仓库 `kikka1777-afk/ai-shortdrama-studio-app`，用 Vercel 部署成线上网址，API 走通用 OpenAI-compatible 配置。

## 一句话指令

> 把 `D:\Drama Agent` 推到 GitHub 仓库 `kikka1777-afk/ai-shortdrama-studio-app`，然后用 `vercel --prod` 部署。部署后在 Vercel 项目里配置 `AI_API_KEY`、`AI_BASE_URL`、`AI_MODEL`，最后重新部署一次。

## 等价命令行

```powershell
cd "D:\Drama Agent"
git add .
git commit -m "feat: add generic api provider settings"
git push origin master
npx vercel --prod
```

## Vercel 环境变量

推荐直接运行脚本，它会提示输入 key，并同步 `AI_PROVIDER`、`AI_BASE_URL`、`AI_MODEL`、`AI_API_KEY` 后自动重新部署：

```powershell
cd "D:\Drama Agent"
.\scripts\sync-vercel-api-settings.ps1
```

通用 OpenAI-compatible：

```powershell
"你的API_KEY" | npx vercel env add AI_API_KEY production
"https://api.x.ai/v1" | npx vercel env add AI_BASE_URL production
"grok-4.3" | npx vercel env add AI_MODEL production
npx vercel --prod
```

OpenAI 官方：

```powershell
"你的OPENAI_KEY" | npx vercel env add AI_API_KEY production
"https://api.openai.com/v1" | npx vercel env add AI_BASE_URL production
"gpt-4o-mini" | npx vercel env add AI_MODEL production
npx vercel --prod
```

Azure OpenAI：

```powershell
"azure-key" | npx vercel env add AI_API_KEY production
"https://your-resource.openai.azure.com" | npx vercel env add AI_BASE_URL production
"deployment-name" | npx vercel env add AI_MODEL production
"azure" | npx vercel env add AI_PROVIDER production
"2024-08-01-preview" | npx vercel env add AI_API_VERSION production
npx vercel --prod
```

旧变量 `OPENAI_API_KEY`、`OPENAI_BASE_URL` 仍兼容，但后续建议统一用 `AI_*`。
