## 等价命令行（Codex 会自动执行，供参考）

```bash
cd "D:\Drama Agent"

# 1) 初始化 git
git init
git add .
git commit -m "init: AI 短剧工作流 (Writer/Director/Stylist)"

# 2) 建 GitHub 仓库并推送（需要 gh CLI 已登录：gh auth login）
gh repo create ai-shortdrama-studio-app --public --source=. --remote=origin --push

# 3) 部署到 Vercel（需要 vercel CLI 已登录：vercel login）
npx vercel --prod        # 首次会问几个问题，一路默认/回车即可，结束给出网址

# 4) 配置 API Key（环境变量），二选一：
#   方式A 用命令行：
echo "你的OPENAI_KEY" | npx vercel env add OPENAI_API_KEY production
# 如走公司中转/网关，再加 base url：
echo "https://你的中转地址/v1" | npx vercel env add OPENAI_BASE_URL production
#   方式B 用网页：Vercel 项目 → Settings → Environment Variables 手动加

# 5) 让环境变量生效，重新部署
npx vercel --prod
```

## 验证（部署完打开网址）
- 点 **「▶ 跑内置样例」** → 应能看到 Writer/Director/Stylist 三段内置产出（不需要 key）。
- 右上 **「⚙ 设置接口」** 填 key 后，或上面配了环境变量后，点 **「▶ 运行工作流」** → 对自己贴的素材实时生成。
- 报错时弹窗会写明哪一步出错（接口地址 / key / 部署名）。

## 接口说明（generate.js 已支持）
- **OpenAI 兼容**：环境变量 `OPENAI_API_KEY` 必填；`OPENAI_BASE_URL` 选填（不填=OpenAI 官方，可填公司中转 `.../v1`）。
- **Azure OpenAI**：在页面「设置接口」选 Azure，填 Endpoint + 部署名 + api-version；或把 Endpoint 设成 `OPENAI_BASE_URL`。
